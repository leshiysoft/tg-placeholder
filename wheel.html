Пусть есть упорядоченная последовательность челых положительных чисел. Мы знаем, что значения этих чисел не превышают некоторое относительно небольшое N. Нам нужно уметь один раз - O(n) - пробежаться по этой последовательности, и составить индекс, который для каждого значения может сказать в каких позициях это значение появлялось в исходной последовательности.
Существующее решение: std::vector>, где размер вектора = N. Здесь для каждого значения есть список позиций, где оно встречалось.
Задача: найти более эффективную замену структуре std::list.
Проверять будем следующим образом: на входе std::vector - исходная последовательность и N - максимально возможное значение. Пробегаем по последовательности и все пишем в наш индекс - это время записи. Далее для всех значений вычитываем номера строк и считаем от них, допустим, некоторую контрольную сумму - это время чтения. Последовательно, в которой десь опрашиваются значения мы заранее не знаем, поэтому там вектор - O(1) для обращания к нужному списку позиций. Важно чтобы оба: и время чтения и время записи были как можно минимальны. Второй по важности задачей является сокращение объема паамяти, необходимое для хранения индекса.
