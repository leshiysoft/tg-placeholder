Задача: построение эффективного индекса позиций значений

Пусть есть упорядоченная последовательность целых положительных чисел. Мы знаем, что значения этих чисел не превышают некоторое относительно небольшое N.

Нам нужно уметь один раз — за O(n) — пробежаться по этой последовательности и составить индекс, который для каждого значения может сказать, в каких позициях это значение появлялось в исходной последовательности.

Текущее решение

Используется структура: std::vector>, где размер вектора = N. Для каждого значения хранится список позиций, где оно встречалось.

Цель

Найти более эффективную замену структуре std::list.

Способ проверки

  • На входе: std::vector — исходная последовательность и N — максимально возможное значение.
  • Этап записи: пробегаем по последовательности и записываем позиции в индекс — измеряем время записи.
  • Этап чтения: для всех значений извлекаем позиции и считаем, например, контрольную сумму — измеряем время чтения.

Требования

  • Доступ к списку позиций для любого значения должен быть O(1) — поэтому используется вектор.
  • Важно: минимизировать и время записи, и время чтения.
  • Второстепенная цель: сократить объём памяти, необходимый для хранения индекса.
